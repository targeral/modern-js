使用 Rspack#

这篇文档会向你介绍如何在 Builder 中开启 Rspack 构建模式。

什么是 Rspack

Rspack 是一个基于 Rust 的高性能 Web 构建工具，具备与 Webpack 生态系统的互操作性，可以被 Webpack 项目低成本集成，并提供更好的构建性能。

相较于 Webpack，Rspack 的构建性能有明显提升，除了 Rust 带来的语言优势，这也来自于它的并行架构和增量编译等特性。经过 benchmark 验证，Rspack 可以带来 5 ～ 10 倍编译性能的提升。

启用 Rspack#

在 Modern.js 框架中启用#

Modern.js 框架默认使用 webpack 作为打包工具。如果希望启用 Rspack 作为打包工具，我们推荐通过 Modern.js 生成器 方式启用。

当然，你也可以通过下面的步骤来手动接入:

  1. 安装 @modern-js/builder-rspack-provider（版本号需要与 Modern.js 版本号保持一致）。
pnpm add @modern-js/builder-rspack-provider -D

通过 package.json 或 lock 文件，确认 Modern.js 版本号:

package.json
{
  "devDependencies": {
    "@modern-js/app-tools": "2.8.0"
  }
}
  1. 在 Modern.js 配置文件中声明使用 Rspack:
modern.config.ts
import appTools, { defineConfig } from '@modern-js/app-tools';

+ export default defineConfig<'rspack'>({
  plugins: [
    appTools({
+     bundler: 'experimental-rspack',
    }),
  ],
});

在前端框架中接入#

如果你基于 Builder 开发了一个前端框架，接入 rspack-provider 和 接入 webpack-provider 的方式类似，只需要把依赖从 @modern-js/builder-webpack-provider 改为 @modern-js/builder-rspack-provider 即可。

modern.config.ts
import { createBuilder } from '@modern-js/builder';
import { builderRspackProvider } from '@modern-js/builder-rspack-provider';

const provider = builderRspackProvider({
  builderConfig: {
    // some configs
  },
});

从 webpack 迁移到 Rspack#

Builder 旨在消除不同打包工具之间的主要差异，帮助用户以较低成本进行迁移。但由于各个打包工具提供的能力不同，你仍需要了解其中的部分差异。

本文将从 Builder 的角度介绍 webpack 和 Rspack 之间的区别。如果需要了解更深层次的差异，请参考 Rspack 文档。

Builder 配置差异#

目前，Builder 内大部分的配置项已经适配了 Rspack。Builder 所做的工作包括不限于：

  • 对一些原本使用借助 webpack loader / plugin (在 Rspack 中不支持使用) 实现的功能，使用 Rspack 替代方案实现。比如，通过 css/module 替代 css-loader 的 modules 能力。
  • 对 webpack 插件进行验证及封装。如 @svgr/webpack、webpack-bundle-analyzer 等;
  • 对 Rspack 内置能力进行封装。如 css、minify 等;

Source Config#

Builder 中与源代码解析、编译方式相关的配置。

不支持的配置项及能力包括：

  • source.include
  • source.exclude
  • source.moduleScopes

Html Config#

Builder 中与 HTML 有关的配置。

所有 html 下的配置项及能力在 Rspack 内均可使用。

Security Config#

Builder 中与安全有关的配置。

所有 security 下的配置项及能力均不支持在 Rspack 内使用。

Dev Config#

Builder 中与本地开发有关的配置。

所有 dev 下的配置项及能力在 Rspack 内均可使用。

Output Config#

Builder 中与构建产物有关的配置。

不支持的配置项及能力包括：

  • output.disableCssExtract
  • output.disableSourcemap.css
  • output.enableAssetManifest
  • output.enableCssModuleTSDeclaration
  • output.legalComments
  • output.enableInlineScripts
  • tsChecker 能力
    • output.disableTsChecker

Experiments Config#

Builder 中的一些实验性配置。

所有 experiments 下的配置项及能力均不支持在 Rspack 内使用。

Performance Config#

Builder 中与性能有关的配置。

不支持的配置项及能力包括：

  • performance.profile
  • performance.removeMomentLocale
  • performance.buildCache

Tools Config#

Builder 中与底层工具有关的配置。

不支持的配置项包括：

  • tools.terser
  • tools.cssExtract
  • tools.cssLoader
  • tools.devServer.devMiddleware
  • tools.inspector
  • tools.minifyCss
  • tools.pug
  • tools.styleLoader
  • tools.styledComponents
  • tools.tsLoader
  • tools.tsChecker
  • tools.webpack
  • tools.webpackChain
TIP

除上述配置外，一些已支持的配置可能存在能力差异，请参考各配置 API 获取具体差异信息。

从 webpackChain 迁移到 bundlerChain#

Builder 支持通过 bundlerChain 来修改 Rspack config。通过 bundlerChain 修改的配置，在 webpack 和 Rspack 构建时均可生效。

TIP

bundlerChain 只为 Rspack & webpack 配置提供一致的 api 修改方式，实际 plugin / loader 是否可用，需要看 Rspack / webpack 实际支持情况。

CHAIN_ID 差异#

因为 webpack 和 rspack 的一些实现差异，所以在规则配置上会有些不同。

可通过 tools.bundlerChain#CHAIN_ID 查看所有在 Rspack & webpack 中都支持的规则。

修改 Rspack 配置#

BundlerChain 只支持修改 Rspack & webpack 交集部分，更多 Rspack 独有功能，需要通过 tools.rspack 修改。

在修改 Rspack 配置之前，你可能需要了解 Rspack 和 Webpack 的配置兼容性。

export default {
  tools: {
    rspack: (config, { env }) => {
      if (env === 'development') {
        config.devtool = 'cheap-module-eval-source-map';
      }
      return config;
    },
  },
};

关于 Rspack 的详细配置信息，请参考 Rspack 官网。
